# Get started

Silen turns a folder of Markdown and MDX files into a complete static website.
It uses React for the component model, Vite for the development and build
pipeline, and MDX for content that can grow beyond plain Markdown.

## Install

Use Node.js `^20.19.0 || >=22.12.0` and add Silen to your project:

```sh
pnpm add -D @aicode-nexus/silen
```

## Create the site

Add `docs/.silen/config.ts`:

```ts
import { defineConfig } from '@aicode-nexus/silen'

export default defineConfig({
  title: 'My documentation',
})
```

Then add `docs/index.mdx`:

```mdx
# Hello, Silen

Your documentation starts here.
```

An AI can follow the same supported path without guessing Silen APIs. Point it
at `@aicode-nexus/silen/agent/manifest.json` in the installed package, then ask
it to follow `agent/tasks/create-site.md`. The task links back to the versioned
config and CLI contract instead of duplicating those definitions here.

## Develop and build

```sh
pnpm silen dev docs
pnpm silen build docs
pnpm silen preview docs
```

The production output is written to `docs/.silen/dist` by default. Configure a
`base` when the site is hosted below a subpath, such as GitHub Pages project
sites.

## Add site analytics

Analytics is a site-level feature, so it works with the default theme and custom
themes. Add Google Analytics, Baidu Analytics, or both:

```ts
export default defineConfig({
  analytics: [
    { provider: 'google', id: 'G-XXXXXXXXXX' },
    { provider: 'baidu', id: 'your-baidu-site-id' },
  ],
})
```

Scripts are included only in production builds. Silen reports the initial page
and client-side route changes, while ignoring hash-only navigation. Google
Analytics users should disable automatic history-change pageviews in Enhanced
Measurement to avoid duplicates. For another service, use a `custom` provider
with external or inline scripts and listen for the `silen:pageview` event.
