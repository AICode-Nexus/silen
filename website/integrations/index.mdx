# Integrations

Silen keeps integrations explicit. Plugins run trusted build or client code,
analytics is configured at the site layer, and Ask AI talks only to an endpoint
you operate. None of these features requires importing internal Silen files.

## Plugins

Plugins are ordered factories in `.silen/config.ts`. Use them for MDX
processors, Vite integrations, page metadata, head entries, client setup, or
post-build output:

```ts
import { defineConfig } from '@aicode-nexus/silen'
import readingTime from './plugins/reading-time'

export default defineConfig({
  plugins: [[readingTime, { wordsPerMinute: 250 }]],
})
```

Every plugin has a `name`; add an `id` for multiple instances. Hooks execute in
configuration order and failures report the plugin identity plus hook. Read
the complete [plugin lifecycle](/silen/guide/plugins).

## Analytics

Analytics is theme-independent and production-only. Configure Google, Baidu,
or an ordered custom provider:

```ts
analytics: [
  { provider: 'google', id: 'G-XXXXXXXXXX' },
  { provider: 'baidu', id: 'site-id' },
]
```

Silen reports the initial page and later client-route changes while ignoring
hash-only navigation. Google users should disable automatic history-change
pageviews in Enhanced Measurement to avoid duplicates. A custom provider can
load external or inline scripts and listen for `silen:pageview`. Scripts are
trusted public code; never embed credentials.

## Ask AI

Ask AI is an optional theme endpoint, not a bundled model provider:

```ts
themeConfig: {
  ai: { endpoint: '/api/ask' },
}
```

The browser posts the current route, optional selected text, and user/assistant
messages. Your server authenticates the request and streams NDJSON events of
type `text`, `citation`, or `error`. Keep provider keys and raw upstream errors
on the server. With no endpoint, Silen emits no Ask AI control or client bundle.

For deterministic, provider-free AI access, use the [generated artifacts](/silen/ai/)
and [local MCP workspace](/silen/ai/local-workspace-mcp/).
